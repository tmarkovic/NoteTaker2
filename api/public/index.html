/api/login POST

Det direkta POST anropet till /api/login tar ett JSON objekt innehållande användarnamn och lösenord ifall anvädaren finns registrerad och lösenordet är korrekt så ges en token i svar som bekräftar att användaren är inloggad. EX data: {"username":"martin","password":"apanssonson"}

/api/users POST

Det direkta POST anropet till /api/users görs för att registrera en ny användare. Datan som behöver skickas med är ett JSON object enligt följande: {"username":"martin","password":"apanssonson","passwordConfirmation":"apanssonson"} Om skapandet lyckades så ges svaret 201(CREATED). Vid fel: 401 (Unauthorized).

/api/users/check-availability/"username" GET

Används för att kontrollera om ett användarnamn är ledigt/giltigt eller inte. Används: http://localhost:3000/api/users/check-availability/martin

Kan ge svaren: {"username":"martin","available":true} eller {"username":"martin","available":false}

/api/notes GET

Ger en lista på användarens alla notes. Kräver att användaren skickar med en giltig inloggningstoken i headern: authentication-token

Exempel på svar: [{"id":"032c2b99-1ab3-425d-b22b-833a940f3c1b","title":"hej","_sourceData":{"id":"032c2b99-1ab3-425d-b22b-833a940f3c1b","title":"hej"}}]

/api/notes POST

Skapar en ny note POST datan som måste skickas med behöver ha följande utseende:
{"noteData":{"title":"myTitel","text":"myText"}}

Kräver att användaren skickar med en giltig inloggningstoken i headern: authentication-token

Exempel på svar: {"id":"032c2b99-1ab3-425d-b22b-833a940f3c1b","userId":"6348b8d1-d153-47a9-b1e3-d94da7d0da1a","title":"hej","text":"sds","_sourceData":{"title":"hej","text":"sds","userId":"6348b8d1-d153-47a9-b1e3-d94da7d0da1a","id":"032c2b99-1ab3-425d-b22b-833a940f3c1b"}}
